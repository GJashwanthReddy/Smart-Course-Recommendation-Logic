AI Concepts & Logic

Core Specifications Mapping

This application demonstrates how fundamental Artificial Intelligence concepts solve the real-world problem of personalized academic course planning.

Problem Formulation & Knowledge Representation

We formulate academic planning as a state-space planning problem:

  • State ($S$): A tuple containing completed_courses (set), acquired_skills (set), current_cgpa (float), and total_credits (int).
  • Action ($A$): Taking an available course $C_i$ whose prerequisites are met.
  • Transition Model $T(S, A) \rightarrow S'$: Adds the course code to completed list, adds its skills to the acquired set, increments total credits, and simulates updated CGPA based on interest and difficulty.
  • Cost Model $C(S, A, S')$: The search cost is represented by the course's difficulty level (float 1.0 to 5.0).
  • Goal ($G$): A set of required skills corresponding to career goal profiles. The goal is satisfied when $G \subseteq S.\text{acquired\_skills}$.

Graph/State-Space Search Algorithms

We implement and compare four algorithms to find the sequence of courses required to transition from the initial state to the goal state:

  • BFS: Finds the path with the minimum number of courses (guarantees shortest path length).
  • DFS: Explores deep paths quickly. It is not cost-optimal or length-optimal, but might expand fewer nodes in dense graphs.
  • UCS (Dijkstra): Expands nodes in order of cumulative difficulty, guaranteeing the path with the lowest overall cost (minimum cumulative difficulty).
  • A* Search: Uses a heuristic function to guide search: $f(n) = g(n) + h(n)$.
Admissible and Consistent Heuristic Equation
h(n) = ( |missing_skills(n)| / max_skills_in_dataset ) * min_difficulty_cost

Proof of Admissibility: Since each course can add at most the maximum skills taught by any single course, and costs at least the minimum difficulty cost in our database, the heuristic never overestimates the cost to reach the goal. Thus, $h(n)$ is admissible ($h(n) \le h^*(n)$) and consistent, guaranteeing the optimal path under A* search.

Constraint Satisfaction Problems (CSP)

To organize courses across semesters, we model the curriculum layout as a CSP:

  • Variables ($X$): The set of recommended courses to schedule.
  • Domains ($D$): Semester numbers $\{1, 2, 3, 4\}$.
  • Constraints ($C$):
    1. Prerequisites Constraint: If $C_a$ is a prerequisite for $C_b$, then $Semester(C_a) < Semester(C_b)$.
    2. Credit Capacity Constraint: Total credits scheduled in any semester $S_j \le 8$ credits.
    3. Course Count Limit: At most 3 courses scheduled per semester.

The solver runs backtracking search. It prioritizes variables using the Minimum Remaining Values (MRV) heuristic and uses the Degree Heuristic to break ties (prioritizing courses with the highest number of dependent prerequisites). It performs Forward Checking domain propagation to prune semester choices that violate credit capacities or prerequisite sequences, backtracking immediately upon detecting empty domains.

Decision-Making & Utility Ranking

We evaluate candidate courses using Multi-Attribute Utility Theory (MAUT). The utility $U(C)$ of a course $C$ is a weighted sum of four sub-utilities:

MAUT Utility Equation
U(C) = 0.40 * U_interest(C) + 0.25 * U_skills(C) + 0.25 * U_career(C) + 0.10 * U_difficulty(C)
  • $U_{interest}$ (40%): Overlap between student's core interests and course keywords. Favors interest matching first.
  • $U_{skills}$ (25%): Overlap between course outcomes and skills desired.
  • $U_{career}$ (25%): Alignment of course topics with career objectives.
  • $U_{difficulty}$ (10%): Aligned with student CGPA (high CGPA prefers challenging courses; lower CGPA prefers easier ones).

We also model course tracking as a 2-player game where the student (MAX) maximizes interest utility and the advisor (MIN) balances academic difficulty. A 3-depth Minimax search tree with Alpha-Beta Pruning simulates the optimal track choice.

Uncertainty Handling via Bayesian Networks

Course outcomes are uncertain. We calculate the probability of successful course completion, $P(\text{Success}=1)$, using a Bayesian belief network:

  • Parent nodes: Interest Fit ($I$), Student Academic Background ($B$), and Course Difficulty ($D$).
  • Prior Probabilities: Derived dynamically from student CGPA, prerequisites, and interest keywords.
  • Conditional Probability Table: Specifies completion rates based on parameters (e.g. $P(S=1 | I=1, B=1, D=0) = 0.95$, while $P(S=1 | I=0, B=0, D=1) = 0.10$).

Exact inference is performed by joint probability enumeration over the variables:

P(Success=1) = Σi,b,d P(Success=1 | I=i, B=b, D=d) * P(I=i) * P(B=b) * P(D=d)

Integrated XAI Recommendation Pipeline

The recommendation engine combines all components into a single pipeline. For any profile:

  1. Formulate student state and goal requirements.
  2. Evaluate utilities to identify and sort candidates.
  3. Evaluate Bayesian networks to compute confidence percentages.
  4. Compare search algorithms to determine course paths.
  5. Solve constraint planning to schedule semesters.
  6. Formulate natural language justifications (traces) explaining the logic behind recommendations.

This results in a complete, explainable recommendation pipeline ready for academic course selection advisor deployment.
